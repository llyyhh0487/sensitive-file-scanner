# 敏感文件扫描工具

## 📖 项目简介

一款基于 **Python + DeepSeek AI** 的自动化敏感文件扫描工具。输入目标网站 URL 和敏感路径字典，即可批量探测 `.git/config`、`.env`、`数据库备份` 等敏感文件是否暴露在公网上，并通过 **AI 语义分析** 精准区分真实漏洞与"返回 200 但页面不存在的"误报。

---

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 **批量路径扫描** | 支持自定义敏感路径字典，并发扫描数千条路径 |
| 🧠 **AI 语义校验** | 调用 DeepSeek API，让大模型判断页面是真实敏感文件还是"假 200" |
| 🚫 **多层误报过滤** | 5 层筛选机制，大幅降低误报率 |
| 🔄 **重定向跟踪** | 自动跟随 301/302 重定向链，分析最终目标页面 |
| 📂 **双重扫描模式** | 同时扫描子目录 + 根目录，不遗漏任何路径组合 |
| 🌐 **目录穿越探测** | 支持路径穿越（Path Traversal）攻击探测 |
| 🖥️ **图形化界面** | PyQt5 桌面 GUI，操作简单直观 |
| 📊 **报告导出** | 支持 Markdown 格式扫描报告 |

## 🛡️ 5 层筛选对比机制

| 层级 | 机制 | 说明 |
|------|------|------|
| **① 状态码过滤** | HTTP 状态码判断 | 只有 200 进入分析，403/401/超时直接分类 |
| **② 本地关键词匹配** | 22 个 404 特征词 | 命中 ≥2 个（如"not found"、"404"）→ 判为误报 |
| **③ 404 指纹比对** | 哈希对比 | 扫前获取站点真实 404 页面，与每个 200 页面对比 |
| **④ DeepSeek AI 分析** | 大模型语义理解 | 发送页面内容前 1500 字符，让 AI 判断真实漏洞 |
| **⑤ 降级本地规则** | 27 个敏感关键词 | AI 不可用时，本地匹配 password、api_key 等关键词 |

**设计理念：** 前三层快速过滤大量误报，后两层深度判断保证准确率。

## 🔄 重定向智能跟踪

### 为什么需要重定向跟踪？

许多现代 Web 应用会对未认证用户的重定向，例如：

- 访问 `/admin` → **302** 重定向到 `/login`（登录页）
- 访问 `/wp-admin` → **301** 重定向到 `/wp-login.php`（WordPress）
- 访问 `/api/private` → **302** 重定向到 `/403`（权限不足页）

如果直接忽略这些 301/302 响应，就会**遗漏大量暴露的敏感路径**。重定向跟踪的核心价值在于：**发现那些虽然重定向了、但最终目标页仍然泄露敏感信息的情况**。

---

### 跟踪原理（代码实现）

工具通过 `_follow_redirect()` 方法实现递归式重定向链跟踪，核心流程如下：

```
┌─────────────────────────────────────────────────────┐
│ 1. 请求路径返回 301/302/307/308                     │
│    └→ 从 response.headers["Location"] 提取跳转目标  │
├─────────────────────────────────────────────────────┤
│ 2. URL 拼接                                         │
│    └→ urljoin(当前URL, Location头)                  │
│    └→ 处理相对路径 (/login → https://x.com/login)   │
│    └→ 处理绝对路径 (http://... 保持不变)            │
├─────────────────────────────────────────────────────┤
│ 3. 安全检查 (4 层过滤)                              │
│    ├─ 去重检查: if next_url in self._seen_urls      │
│    ├─ 同域限制: netloc 必须与原始目标一致            │
│    ├─ 跳数限制: 最多 hops < max_hops (默认3)        │
│    └─ Location为空: 无法继续则中止                  │
├─────────────────────────────────────────────────────┤
│ 4. 递归跟随                                         │
│    └→ 对 next_url 发起新请求 (allow_redirects=False)│
│    └→ 不自动跟随的原因是：需要手动控制每一步         │
│    └→ 记录完整链路: chain = [原始 → 中间 → ... → 最终]│
├─────────────────────────────────────────────────────┤
│ 5. 终止条件判断                                     │
│    ├─ 状态码 200     → 到达终点，进入 AI 分析       │
│    ├─ 状态码 401/403 → 需要认证，记录并返回          │
│    ├─ 状态码 3xx     → 继续递归 (下一跳)             │
│    └─ 其他状态码     → 中止跟踪                      │
├─────────────────────────────────────────────────────┤
│ 6. AI 语义分析最终目标                               │
│    └→ 将最终 200 页面内容 (前1500字符) 发送给 AI    │
│    └→ AI 判断：是真正的敏感页面 or 通用404不存在页   │
└─────────────────────────────────────────────────────┘
```

---

### 关键设计细节

| 设计项 | 策略 | 原因 |
|--------|------|------|
| **手动递归** | `allow_redirects=False` + 手动跟随 | 需要记录每一步的 URL 链、状态码、响应内容，`requests` 自动重定向会丢失中间步骤 |
| **同域限制** | `urlparse(next_url).netloc != urlparse(self.url).netloc` 时中止 | 防止被重定向到钓鱼/恶意网站，也避免跟踪 CDN/OAuth 等第三方跳转 |
| **最多 3 跳** | `max_hops=3` | 防止重定向死循环（A→B→C→A…），同时避免拉长扫描时间 |
| **URL 去重** | `self._seen_urls` 集合去重 | 同一扫描会话中已请求过的 URL 不再重复，防止回环重定向 |
| **相对路径处理** | `urljoin(current_url, location)` | 正确处理 `/login`、`./dashboard`、`../admin` 等相对路径 |

---

### 风险等级自动升级

当重定向最终到达一个 AI 判定为低风险的页面时，工具会**自动提升风险等级**：

| 原始风险 | 升级后 | 原因 |
|----------|--------|------|
| `low` → | `medium` | 敏感路径虽然重定向了，但仍然可达 |
| `info` → | `medium` | 重定向后的页面仍需引起注意 |

### 实际效果示例

```
请求: https://target.com/.git/config
  └→ 302 → Location: /login
      └→ 自动拼接: https://target.com/login
          └→ 200 (页面含 "403 Forbidden" → AI 判为误报)
              ✗ 过滤：虽然是真实页面，但内容是权限不足

  └→ 302 → Location: /admin/login
      └→ 自动拼接: https://target.com/admin/login

请求: https://target.com/backup
  └→ 301 → Location: /files/backup/
      └→ 自动拼接: https://target.com/files/backup/
          └→ 200 (目录列表 + "database_backup.sql")
              ✓ 高危：重定向后直接暴露了数据库备份文件！
```

## 🚀 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 命令行扫描

```bash
python sensitive_scanner.py --url https://example.com --dict dicts/sensitive_paths.txt
```

### 启动图形界面

```bash
python sensitive_scanner.py --gui
```

### 测试演示

```bash
# 终端1：启动本地测试服务器
python test_target.py

# 终端2：运行扫描器
python sensitive_scanner.py --url http://127.0.0.1:5000 --dict dicts/sensitive_paths.txt
```

## ⚙️ 核心参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--url` | 目标 URL | 必填 |
| `--dict` | 敏感路径字典文件 | `dicts/sensitive_paths.txt` |
| `--concurrency` | 并发线程数 | 30 |
| `--timeout` | 请求超时（秒） | 8 |
| `--no-ai` | 关闭 AI 校验 | 开启 |
| `--no-root` | 关闭根目录扫描 | 开启 |
| `--no-traversal` | 关闭目录穿越扫描 | 开启 |
| `--output` | 报告输出路径 | 自动生成 |
