# 🔍 敏感文件扫描工具

> 基于 **Python + DeepSeek AI** 的自动化敏感文件扫描工具

## 📖 项目简介

一款基于 Python + DeepSeek AI 的自动化敏感文件扫描工具。输入目标网站 URL 和敏感路径字典，即可批量探测 `.git/config`、`.env`、`数据库备份` 等敏感文件是否暴露在公网上，并通过 **AI 语义分析** 精准区分真实漏洞与"返回 200 但页面不存在的"误报。

---

## ✨ 核心功能

| 功能 | 说明 |
|------|------|
| 🔍 **批量路径扫描** | 支持自定义敏感路径字典，并发扫描数千条路径 |
| 🧠 **AI 语义校验** | 调用 DeepSeek API，让大模型判断页面是真实敏感文件还是"假 200" |
| 🚫 **多层误报过滤** | 5 层筛选机制，大幅降低误报率 |
| 🔄 **重定向跟踪** | 自动跟随 301/302 重定向链，分析最终目标页面 |
| 📂 **双重扫描模式** | 同时扫描子目录 + 根目录，不遗漏任何路径组合 |
| 🌐 **目录穿越探测** | 支持路径穿越（Path Traversal）攻击探测 |
| 🖥️ **图形化界面** | PyQt5 桌面 GUI，操作简单直观 |
| 📊 **报告导出** | 支持 Markdown 格式扫描报告 |

## 🏗️ 技术架构

```
用户输入 URL + 字典 → 图形界面 (PyQt5)
                          ↓
                   核心扫描引擎 (asyncio 异步并发)
                          ↓
         ┌────────────────┼────────────────┐
         ↓                ↓                 ↓
   HTTP 请求层        误报过滤层        AI 校验层
 (requests.Session)  (5层筛选机制)   (DeepSeek API)
         ↓                ↓                 ↓
         └────────────────┼────────────────┘
                          ↓
                    漏洞结果输出
                   (真实敏感文件列表)
```

---

## 🛡️ 5 层筛选对比机制

| 层级 | 机制 | 说明 |
|------|------|------|
| **① 状态码过滤** | HTTP 状态码判断 | 只有 200 进入分析，403/401/超时直接分类 |
| **② 本地关键词匹配** | 22 个 404 特征词 | 命中 ≥2 个（如"not found"、"404"）→ 判为误报 |
| **③ 404 指纹比对** | 哈希对比 | 扫前获取站点真实 404 页面，与每个 200 页面对比 |
| **④ DeepSeek AI 分析** | 大模型语义理解 | 发送页面内容前 1500 字符，让 AI 判断真实漏洞 |
| **⑤ 降级本地规则** | 27 个敏感关键词 | AI 不可用时，本地匹配 password、api_key 等关键词 |

**设计理念：** 前三层快速过滤大量误报，后两层深度判断保证准确率。

## 🔄 重定向智能跟踪

### 为什么需要重定向跟踪？

许多现代 Web 应用会对未认证用户的重定向，例如：

- 访问 `/admin` → **302** 重定向到 `/login`（登录页）
- 访问 `/wp-admin` → **301** 重定向到 `/wp-login.php`（WordPress）
- 访问 `/api/private` → **302** 重定向到 `/403`（权限不足页）

如果直接忽略这些 301/302 响应，就会**遗漏大量暴露的敏感路径**。重定向跟踪的核心价值在于：**发现那些虽然重定向了、但最终目标页仍然泄露敏感信息的情况**。

## 🚀 快速开始

### 安装依赖

```bash
pip install -r requirements.txt
```

### 命令行扫描

```bash
python sensitive_scanner.py --url https://example.com --dict dicts/sensitive_paths.txt
```

### 启动图形界面

```bash
python sensitive_scanner.py --gui
```

### 测试演示

```bash
# 终端1：启动本地测试服务器
python test_target.py

# 终端2：运行扫描器
python sensitive_scanner.py --url http://127.0.0.1:5000 --dict dicts/sensitive_paths.txt
```

## ⚙️ 核心参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--url` | 目标 URL | 必填 |
| `--dict` | 敏感路径字典文件 | `dicts/sensitive_paths.txt` |
| `--concurrency` | 并发线程数 | 30 |
| `--timeout` | 请求超时（秒） | 8 |
| `--no-ai` | 关闭 AI 校验 | 开启 |
| `--no-root` | 关闭根目录扫描 | 开启 |
| `--no-traversal` | 关闭目录穿越扫描 | 开启 |
| `--output` | 报告输出路径 | 自动生成 |
